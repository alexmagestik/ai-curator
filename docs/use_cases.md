# Учебные сценарии (демо для заказчика)

Ниже — три готовых сценария, которые можно воспроизвести в Streamlit после `index_documents.py`. Они показывают, как ассистент сочетает RAG, LMS и эскалацию к человеку.

---

## Сценарий 1. Новичок не понимает тему и уточняет дедлайн

**Персона:** студент первого модуля, уровень *Beginner* (определяется автоматически).

**Цель:** показать связку «учёба → организация» в одном диалоге.

| Шаг | Вопрос студента | Ожидаемое поведение |
|---|---|---|
| 1 | «Зачем нужно виртуальное окружение в Python? Я только начинаю курс.» | **RAG** — простое объяснение, источник `module_01`, `lecture_01.odt`. Статус: зелёный — ответ по материалам. |
| 2 | «Когда нужно сдать первую домашнюю работу?» | **LMS** — дата из `assignments.json` (HW №1, 2026-07-10). Статус: организационный ответ из LMS. |
| 3 | «А что именно входит в домашку по модулю 1?» | **LMS** — описание задания из JSON. |

**На что обратить внимание заказчику:**
- уровень сложности ответа адаптируется под beginner;
- источники и фрагменты видны в expander;
- организационный вопрос не «угадывается» из лекций — только LMS.

## Сценарий 2. Студент просит изменить правила курса

**Персона:** студент модуля 2, спешит со сдачей проекта.

**Цель:** показать guardrails и **эскалацию к человеку** в UI.

| Шаг | Вопрос студента | Ожидаемое поведение |
|---|---|---|
| 1 | «Можно перенести дедлайн домашней работы №2 на следующую неделю?» | Жёлтый статус **«Изменение правил курса»** + подсказка обратиться к куратору. Ответ LLM не должен обещать перенос. |
| 2 | «Какую оценку я получу, если сдам с опозданием?» | Статус **«Запрос об оценке»** — куратор не выставляет и не прогнозирует оценки. |
| 3 | «Хочу поговорить с живым куратором по этому вопросу» | Статус **«Запрос живого специалиста»** с рекомендацией эскалации. |

**На что обратить внимание заказчику:**
- UI явно маркирует случаи, где нужен человек;
- аналитика (`query_logs`) сохранит категорию вопроса для отчёта admin.

## Сценарий 3. Подготовка к итоговому проекту (RAG, модуль 3)

**Персона:** студент с техническим бэкграундом, уровень *Advanced*.

**Цель:** показать глубину RAG-ответа и фильтр по модулю.

| Шаг | Действие | Ожидаемое поведение |
|---|---|---|
| 1 | В sidebar выбрать фильтр **module_03** | Поиск ограничен материалами модуля. |
| 2 | «Какие компоненты входят в RAG-пайплайн и как они связаны между собой?» | **RAG** — развёрнутый ответ: эмбеддинги, vector store, retriever, prompt. Несколько фрагментов из `lecture_01.odt`. |
| 3 | «Когда дедлайн итогового проекта?» | **LMS** — 2026-08-01, описание из assignments. |

**На что обратить внимание заказчику:**
- фильтр по модулю работает только для учебных запросов;
- для итогового проекта студент получает и теорию (RAG), и срок (LMS) в одном интерфейсе.

## Сценарии ошибок (для проверки устойчивости)

| Ситуация | Что увидит пользователь |
|---|---|
| Нет `OPENAI_API_KEY` | Красная ошибка на странице чата + инструкция по `.env`. |
| Не выполнена индексация | Жёлтое предупреждение вверху чата. |
| Вопрос вне материалов («Что такое Kubernetes?») | Жёлтый статус «Материалы курса не покрывают вопрос» + эскалация. |
| Ошибка OpenAI (лимит, сеть) | Красная ошибка с расшифровкой + рекомендация обратиться в поддержку. |

---

## Быстрый чеклист для демо заказчику

1. Войти под тестовым пользователем (или admin).
2. Пройти **Сценарий 1** — 3 сообщения подряд.
3. Показать sidebar: уровень, фильтр модуля, блок «Сценарии и эскалация».
4. Пройти **Сценарий 2** — обратить внимание на жёлтые статусы.
5. (Admin) Открыть **Аналитика** — новые записи в `query_logs`.

См. также: [user_guide.md](user_guide.md), [prompts.md](prompts.md).
