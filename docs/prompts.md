# Архитектура промптов и сценариев

## Роль куратора

Системный промпт задаёт роль **поддерживающего образовательного наставника**:

- дружелюбный, профессиональный, мотивирующий тон;
- помощь в понимании материала и следующих шагах обучения;
- обязательная ссылка на источник (модуль + документ).

Файл: [`app/prompts/system_prompt.txt`](../app/prompts/system_prompt.txt)

## Ограничения (guardrails)

| Запрет | Реализация |
|---|---|
| Не выставляет оценки | Явно в system prompt |
| Не меняет расписание | Организационные данные только из LMS; куратор не «переносит» дедлайны |
| Не придумывает факты | «Опирайся только на контекст» + few-shot негативные примеры |
| Не даёт гарантий результатов | В system prompt |
| Сообщает о нехватке данных | Инструкция в RAG- и LMS-промптах |

## Адаптация уровня сложности

Три уровня: **beginner**, **intermediate**, **advanced**.

### Автоматическое определение
Сервис `LevelDetector` после каждого сообщения студента возвращает:

```json
{"level": "intermediate", "confidence": 0.82}
```

Результат сохраняется в `user_profiles` и используется в следующих ответах.

### Инструкции в промпте

| Уровень | Стиль ответа |
|---|---|
| Beginner | Простые объяснения, минимум терминов, примеры |
| Intermediate | Умеренная глубина, пояснение терминов |
| Advanced | Технические детали, профессиональная терминология |

Код: [`app/prompts/loader.py`](../app/prompts/loader.py) → `build_system_prompt(user_level=...)`

## Few-shot обучение

Файл: [`app/prompts/fewshot_examples.json`](../app/prompts/fewshot_examples.json)

- **10 корректных** примеров — вопрос + ответ со ссылкой на источник;
- **10 некорректных** — плохой ответ + объяснение, почему так нельзя.

В промпт включаются по 3 примера каждого типа для экономии токенов.

## Сценарии ответов

```text
                    ┌─────────────────┐
                    │  Вопрос студента │
                    └────────┬────────┘
                             │
                    classify_intent()
                             │
              ┌──────────────┴──────────────┐
              ▼                             ▼
      course_content                  organizational
      (учебный)                    (deadline/schedule/assignment)
              │                             │
              ▼                             ▼
         RAG Pipeline                  LMS Handler
    (ChromaDB + hybrid)            (schedule.json, assignments.json)
              │                             │
              └──────────────┬──────────────┘
                             ▼
                    LLM + system prompt
                    + user level
                    + chat history
                             ▼
                         Ответ
                             │
                    UI: статус (success / info /
                    warning / escalate)
```

### Эскалация и статусы в UI

После каждого ответа Streamlit показывает статус (`app/ui/response_status.py`):

| Статус | Когда | Действие для студента |
|---|---|---|
| success | Ответ найден в RAG или LMS | Читать ответ и источники |
| info | Мало источников (1 фрагмент) | Открыть полный документ модуля |
| warning | Нет данных в материалах / LMS | Переформулировать или эскалация |
| escalate | Оценки, перенос дедлайнов, оплата, «живой куратор» | Обратиться к человеку |

Код отображения: [`app/pages/chat.py`](../app/pages/chat.py).

Демо-сценарии для заказчика: [use_cases.md](use_cases.md).

### Учебные вопросы
Контекст из retrieved chunks → LLM формирует ответ с источниками.

### Организационные вопросы
Данные LMS (расписание, задания, info курса) → LLM отвечает **без** обращения к базе знаний лекций.

## Конфигурируемая LLM

```python
from app.services.llm_factory import get_llm
llm = get_llm()  # LLM_PROVIDER=openai в .env
```

Архитектура позволяет в будущем подключить Anthropic, Gemini, Ollama.

## Редактирование промптов

1. Измените `system_prompt.txt` или `fewshot_examples.json`.
2. Перезапустите Streamlit (кэш `@lru_cache` в loader сбросится при рестарте).
3. Проверьте качество на странице **RAG Debug** (admin).

## A/B тестирование промптов

Прежде чем менять `system_prompt.txt`, можно сравнить новую формулировку с текущей
на странице **A/B тесты** (admin): слепая оценка ответов и метрики Relevance,
Hallucinations, Relative Lift. Подробнее — [ab_testing.md](ab_testing.md).
