# Руководство пользователя

## Первый вход

1. Откройте Streamlit: http://localhost:8501
2. Зарегистрируйтесь (вкладка **Регистрация**) или войдите.
3. Гостевой доступ **не предусмотрен** — нужен аккаунт.

## Чат с куратором

1. Перейдите в раздел **Чат**.
2. Задайте вопрос в поле ввода внизу экрана.
3. Дождитесь ответа — отображаются:
   - текст ответа;
   - тип: **RAG** (учебный) или **LMS** (организационный);
   - источники (модуль, документ);
   - найденные фрагменты (для RAG).

### Уровень подготовки

Уровень (**Beginner / Intermediate / Advanced**) определяется **автоматически** после каждого сообщения. Текущий уровень и уверенность отображаются в боковой панели.

### Фильтр по модулю

В sidebar можно ограничить поиск материалами конкретного модуля (только для учебных вопросов).

### Новый диалог

Кнопка **Новый диалог** в sidebar начинает чистую сессию.

## Мои диалоги

1. Раздел **Мои диалоги** — список всех ваших сессий.
2. Используйте **Поиск** для фильтрации по тексту сообщений.
3. **Продолжить** — загрузит историю и откроет чат.

## Что спрашивать

| Тип | Примеры |
|---|---|
| Учебные | «Что такое REST API?», «Объясни RAG» |
| Дедлайны | «Когда дедлайн домашней работы №1?» |
| Расписание | «Когда следующее занятие?» |
| Задания | «Какие задания нужно сдать?» |

## Статусы ответа и эскалация

Под каждым ответом ассистента отображается статус:

| Цвет | Значение |
|---|---|
| Зелёный | Ответ найден в материалах или LMS |
| Синий | Ответ есть, но опирается на один источник — стоит открыть полный документ |
| Жёлтый | Недостаточно данных **или** нужен живой куратор (оценки, перенос дедлайна, оплата) |

При жёлтом статусе следуйте рекомендации в подписи — обратитесь к куратору курса или в поддержку платформы.

При ошибке API (красный блок) проверьте ключ OpenAI и повторите запрос.

Подробные демо-сценарии для заказчика: [use_cases.md](use_cases.md).

## Демо-сценарии (кратко)

1. **Новичок:** «Зачем виртуальное окружение?» → RAG; затем «Когда сдача HW1?» → LMS.
2. **Эскалация:** «Перенесите дедлайн HW2» → жёлтый статус, нужен человек.
3. **Итоговый проект:** фильтр `module_03` + «Компоненты RAG-пайплайна?» → развёрнутый RAG-ответ.

## Для администратора

Admin создаётся при инициализации БД (`INIT_ADMIN_EMAIL` / `INIT_ADMIN_PASSWORD` в `.env`).

Дополнительные разделы:

| Раздел | Действия |
|---|---|
| **База знаний** | Просмотр документов, инкрементальная/полная переиндексация |
| **RAG Debug** | Тест retrieval без сохранения в историю чата |
| **Аналитика** | Метрики, графики, CSV-экспорт, просмотр логов |

## Выход

Кнопка **Выйти** в sidebar (на странице Чат или Мои диалоги).
